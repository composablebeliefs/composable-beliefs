**What actually breaks.** Hooks *do* run in remote sessions - Claude Code on the web executes them (SessionStart hooks are an advertised feature of it). Your Stop hook fails here for three separable reasons:

1. **Registration is local.** No committed `.claude/settings.json` wires the hook up; it's presumably registered in your local settings, which don't travel with the clone.
2. **The path is local.** `THREADS_DIR` hardcodes `/Users/mark/...` - the limitation thread-repo-binding already records.
3. **The persistence model is local.** This is the deep one. Locally, "write to a gitignored dir every turn" *is* persistence - the disk survives the session. Remotely, the working tree dies with the container, so a hook that only writes files achieves nothing durable. Remote persistence is push-only.

The first two are trivial fixes; the third is the real design question.

**The workable substitute, in tiers:**

- **Tier 1 - make the existing hook environment-agnostic.** Commit the hook registration in the repo's `.claude/settings.json` and derive the output dir from `$CLAUDE_PROJECT_DIR` (hooks receive it) instead of the hardcoded path. Same hook then runs identically local and remote; remotely the render accumulates in the working tree. This also fixes thread-repo-binding's complaint in one move.

- **Tier 2 - close the persistence gap at session end.** Since remote persistence is push-only, the convention becomes: the thread render is committed with the session's final push. That's exactly transcript-format's target shape anyway - hook owns the live body, `/end` finalizes and commits. The `/end` step can be a skill or a directive; and the Stop hook itself can emit a reminder into my context when the render sits uncommitted, which converts "remember to commit the thread" from purely procedural (the a386 failure mode) into prompted. Worth noting: the raw jsonl is also available *inside* the container (`transcript_path`), so the persist-raw decision is servable remotely too - copy the jsonl in and commit it (LFS, per the lean) in the same final push.

- **Tier 3 - true crash-safety remotely, probably over-engineered.** A hook that commits and pushes every turn to a dedicated ref (`threads/<session-id>`, never the working branch). It's the only way to replicate the local crash-safety property, but it costs network per turn, sprays refs, and entangles hook pushes with the agent's own git operations. I'd skip it: the crash-safety rationale barely translates - remote sessions are retained and resumable on the claude.ai side, so a mid-session container death doesn't lose the conversation the way a local crash could.
